# Подробный план развития проекта мобильного робота (LOOI Robot Mobile MVP)

Этот файл содержит детальный технический план доработок, которые мы наметили сделать в будущем. Он лежит здесь, чтобы всегда быть под рукой в корне проекта.

---

## 1. Добавление памяти диалога (Контекст ИИ)
Сейчас робот страдает склерозом: он видит только последнюю фразу пользователя. Мы научим его помнить контекст беседы.

### Что нужно сделать:
* **Хранение истории**:
  Внутри хука [useRobotControl.ts](src/hooks/useRobotControl.ts) создадим состояние `chatHistory`:
  ```typescript
  const [chatHistory, setChatHistory] = useState<Array<{ role: 'user' | 'assistant'; content: string }>>([]);
  ```
* **Пакетная отправка сообщений**:
  При формировании запроса к OpenRouter склеиваем:
  1. Системный промпт (правила поведения робота и описание инструментов).
  2. Предыдущие реплики из `chatHistory` (например, последние 6-8 сообщений, чтобы экономить токены и не превышать контекст).
  3. Новую реплику пользователя.
* **Обновление истории**:
  * Когда пользователь отправляет фразу $\rightarrow$ добавляем её в `chatHistory` с ролью `user`.
  * Когда ИИ возвращает текстовый ответ $\rightarrow$ добавляем его в `chatHistory` с ролью `assistant`.
* **Очистка контекста**:
  Добавим в настройки [ConfigOverlay.tsx](src/components/ConfigOverlay.tsx) или на главный экран кнопку **"Сбросить память"** (очистить историю диалога), чтобы робот мог начать общение с чистого листа.

## 2. Динамический выбор модели ИИ (без хардкода)
Вместо зашитой модели `qwen/qwen-2.5-72b-instruct` мы дадим пользователю возможность выбирать мозг робота через интерфейс настроек.

### Что нужно сделать:
* **Расширение конфигурации**:
  В хуке [useConfig.ts](src/hooks/useConfig.ts) добавим поля `aiModel` и `aiModelInput` (с дефолтным значением `qwen/qwen-2.5-72b-instruct`).
* **Сохранение в хранилище**:
  Сохраняем выбранную модель в `AsyncStorage` под ключом `OPENROUTER_AI_MODEL`.
* **UI в настройках**:
  В [ConfigOverlay.tsx](src/components/ConfigOverlay.tsx) добавим новое поле ввода для названия модели ИИ (или выпадающий список популярных вариантов: Qwen, Llama 3, Gemini Flash).
* **Динамический запрос**:
  В [useRobotControl.ts](src/hooks/useRobotControl.ts) подставим сохраненное значение модели в тело POST-запроса:
  ```json
  "model": apiKey ? aiModel : "qwen/qwen-2.5-72b-instruct"
  ```

## 3. Озвучка реплик робота (Text-to-Speech)
Робот должен говорить голосом, а не заставлять пользователя читать с экрана смартфона.

### Что нужно сделать:
* **Добавление зависимости**:
  Установить официальный модуль озвучки от Expo:
  ```bash
  npx expo install expo-speech
  ```
* **Реализация озвучки**:
  В [useRobotControl.ts](src/hooks/useRobotControl.ts) импортируем `* as Speech` из `expo-speech` и при получении текстового ответа от ИИ вызываем:
  ```typescript
  Speech.speak(message.content, {
    language: 'ru-RU',
    pitch: 1.1, // Чуть повысим тон для более "роботизированного/милого" звучания
    rate: 1.0,
  });
  ```
* **Прерывание озвучки**:
  If робот говорит, но пользователь начинает диктовать новую команду или нажимает кнопку ручного управления, мы должны мгновенно вызвать `Speech.stop()`, чтобы робот не перебивал сам себя и не бубнил старую фразу поверх новой задачи.

## 4. Обратная связь от датчиков ESP32 (Двустороннее общение)
Робот должен реагировать на столкновения и препятствия, меняя эмоцию и жалуясь пользователю.

### Что нужно сделать:
* **Протокол ответа ESP32**:
  При отправке POST-запроса движения на `http://<ESP_IP>/api/move`, ESP32 в ответ присылает не просто `HTTP 200 OK`, а JSON со статусом датчиков:
  ```json
  {
    "status": "success",
    "obstacle_detected": false,
    "battery_mv": 3800
  }
  ```
  Или в случае аварии (уперся в стену по датчику расстояния/бамперу):
  ```json
  {
    "status": "blocked",
    "obstacle_detected": true,
    "reason": "front_sensor"
  }
  ```
* **Реакция на стороне смартфона**:
  * Если `obstacle_detected` равен `true`, прерываем таймер движения в [useRobotControl.ts](src/hooks/useRobotControl.ts) и переключаем эмоцию в состояние `stop` (красный цвет глаз).
  * Можно добавить новую эмоцию `scared` (испуг с расширенными зрачками).
* **Событийный промпт для ИИ**:
  При обнаружении препятствия приложение автоматически добавляет в историю диалога системное сообщение:
  `[Система: Движение заблокировано датчиком препятствия! Спереди стена. Жалуйся пользователю и проси убрать препятствие]`
  И автоматически триггерит запрос к ИИ, чтобы робот сам заговорил и озвучил проблему своим синтезированным голосом.

## 5. Режим непрерывного hands-free диалога (Conversation Mode)
Чтобы с роботом можно было общаться без необходимости постоянно зажимать кнопку микрофона на экране.

### Что нужно сделать:
* **Автоматическое определение тишины**:
  Использовать функцию Voice Activity Detection (VAD), которая встроена в `expo-speech-recognition`, чтобы распознавание автоматически завершалось, когда пользователь замолчал.
* **Цепочка обратной связи**:
  Создадим триггер автоматического возобновления записи:
  1. Пользователь один раз активирует режим диалога.
  2. Микрофон открывается $\rightarrow$ пользователь говорит фразы (например: *"Привет, как дела?"*).
  3. Как только пользователь замолчал, распознанный текст отправляется ИИ.
  4. ИИ генерирует ответ $\rightarrow$ срабатывает Text-to-Speech (озвучка реплики).
  5. Как только озвучка робота завершается (`Speech.speak` завершает воспроизведение) $\rightarrow$ приложение автоматически снова включает микрофон для следующей реплики.
* **Индикатор тишины**:
  Визуально подсвечивать зрачки или рамку экрана в режиме ожидания ответа пользователя (например, пульсирующий зеленый или синий микрофон), чтобы было понятно, когда робот слушает, а когда говорит.
